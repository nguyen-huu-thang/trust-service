KEY SERVICE – PUBLIC KEY DISTRIBUTION

1. Hiện tại (MVP)

* Sử dụng gRPC cho toàn bộ giao tiếp.
* Identity Service:
  → lấy private key từ Key Service.
* Các service khác:
  → lấy public key qua gRPC.
* Không expose public key ra Internet (không dùng JWKS public endpoint).

2. Nguyên tắc

* Public key vẫn là “public” theo nghĩa cryptography:
  → không cần giữ bí mật.
* Tuy nhiên:
  → chỉ chia sẻ trong phạm vi hệ thống (internal services).
* Đây là:
  → “internal public key distribution”.

3. Lý do chọn thiết kế này

* Giảm attack surface (không public ra Internet).
* Phù hợp với hệ thống microservice nội bộ.
* Đơn giản hóa MVP, tránh over-engineering.

4. Lưu ý

* Key Service không nên là runtime dependency bắt buộc.
* Luôn trả:
  → NEXT + CURRENT + OLD key (để verify JWT không bị lỗi).

5. Hướng mở rộng (tương lai)

* Có thể thêm JWKS endpoint public nếu cần external access.
* Có thể dùng hybrid:
  → internal: gRPC
  → external: JWKS
